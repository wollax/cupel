# T02: Implement IEquatable on .NET ContextItem with collection-aware equality

**Custom Equals/GetHashCode on ContextItem with SequenceEqual for Tags and element-wise comparison for Metadata, 14 test cases covering ordered tags, metadata nulls, and hash consistency**

## What Happened

Added `IEquatable<ContextItem>` to the sealed record declaration and overrode `Equals(ContextItem?)` with:
- Null/reference-equality short-circuits
- Scalar comparison for Content, Tokens, Kind, Source, Priority, Pinned, OriginalTokens, FutureRelevanceHint, Timestamp
- `Tags.SequenceEqual(other.Tags)` for ordered list comparison
- Private `MetadataEquals` helper using count check + element-wise `TryGetValue` + `object.Equals` for null-safe dispatch

Overrode `GetHashCode()` using `HashCode.Combine` for scalar fields plus pragmatic O(1) collection contributions (Tags.Count + first element, Metadata.Count).

Created 14 test cases in `ContextItemEqualityTests.cs` covering: identical items, different content, same/different tag order, same/different metadata keys/values, null vs non-null metadata values, both-null metadata values, null vs non-null timestamp, different FutureRelevanceHint, null safety, hash consistency, and with-expression deep copy equality.

Updated `PublicAPI.Unshipped.txt` with `Equals(ContextItem?)` and `GetHashCode()` entries.
