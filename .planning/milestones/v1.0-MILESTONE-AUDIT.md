## Remaining Tech Debt

### Phase 09: Custom Scorer Deserialization (Low)

`RegisterScorer(string name, Func<IScorer> factory)` exists and enriches error messages, but custom scorer type names cannot participate in JSON deserialization. The `ScorerType` enum is closed; STJ cannot deserialize unknown enum values. This is acknowledged in the Phase 09 plan as a design limitation. The hook exists for future extensibility.

**Impact:** Consumers using custom scorers must build pipelines programmatically — they cannot round-trip custom scorer policies through JSON.

### Phase 07: Unused ExclusionReason Values (Informational)

`ExclusionReason.ScoredTooLow` and `ExclusionReason.Filtered` are defined in the enum and registered in PublicAPI but are never emitted by the current pipeline. They are reserved for future use by custom slicers or filter stages.

**Impact:** None — enum completeness for extensibility.

### Phase 07: Truncate O(n^2) (Low)

The `OverflowStrategy.Truncate` path uses `List.RemoveAt` in a loop, which is O(n^2) in the worst case. Overflow handling is a rare code path and N is bounded by the item count.

**Impact:** Negligible for expected workloads (<500 items).

### Phase 14: PublicAPI.Unshipped.txt (Resolved)

`ScorerType.Scaled`, `SlicerType.Stream`, `ScorerEntry.InnerScorer`, and `CupelPolicy.StreamBatchSize` promoted to `PublicAPI.Shipped.txt`. `PublicAPI.Unshipped.txt` is now empty.

### Phase 05: Sort Without Trace Event (By Design)

The Sort step between Deduplicate and Slice is an internal implementation detail. It has no `PipelineStage` enum value and emits no trace event. This is intentional — Sort is not a user-facing pipeline stage.

**Impact:** None.

### Total: 4 items across 3 phases — 0 blockers (1 resolved)
