# Milestone v1.0: Core Library

**Status:** SHIPPED 2026-03-14
**Goal:** Ship the complete Cupel context management library — pipeline engine, scoring, slicing, placement, explainability, fluent API, named policies, serialization support, and all NuGet packages.
**Phases:** 1-15
**Total Plans:** 48

## Overview

Cupel v1.0 delivers a .NET context management library for coding agents. Given a set of context items (messages, documents, tool outputs, memory) and a token budget, Cupel determines the optimal context window — maximizing information density while respecting the attention mechanics of any LLM.

## Milestone Summary

**Key Decisions:**
- Content is non-nullable string on ContextItem (simplifies API, maximizes debuggability)
- Ordinal-only scoring invariant (scorers rank, slicers drop, placers position)
- CompositeScorer over scorer DAG (nested composites, ~30 lines vs hundreds)
- Fixed pipeline over middleware (no call-next, no silent-drop)
- IPlacer interface (pluggable, U-shaped as default not mandate)
- ContextBudget is sealed class (not record) to prevent with-expressions bypassing validation
- Zero-allocation discipline on hot paths (for loops with indexer, no LINQ/foreach/closures)

**Issues Resolved:**
- ReservedSlots and EstimationSafetyMarginPercent wired into pipeline (Phase 13)
- ScaledScorer and StreamSlice reachable from policy/JSON/DI (Phase 14)
- DI lifetime divergence fixed with PolicyComponents keyed singleton (Phase 14)
- QuotaSlice promoted to required conformance tier (Phase 15)
- PublicAPI.Unshipped.txt promoted to Shipped.txt

**Technical Debt Remaining:**
- RegisterScorer hook is informational only (STJ closed enum limitation)
- ExclusionReason.ScoredTooLow and .Filtered reserved but never emitted
- Truncate path O(n^2) List.RemoveAt (negligible at <500 items)
- Sort step has no PipelineStage enum value (by design)
