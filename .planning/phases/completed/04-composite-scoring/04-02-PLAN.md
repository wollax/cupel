<objective>
Implement ScaledScorer: a sealed IScorer wrapper that applies min-max normalization to any inner scorer's output, mapping it to the [0, 1] range across the candidate set. Two-pass approach: scan all items for min/max, then normalize the current item's raw score. Degenerate case (all identical) returns 0.5. Must integrate with CompositeScorer's cycle detection — expose the inner scorer for traversal.
</objective>

  <implementation>
    ScaledScorer is a sealed class in namespace Wollax.Cupel.Scoring implementing IScorer.

    Constructor: `public ScaledScorer(IScorer inner)`
    - ArgumentNullException.ThrowIfNull(inner)
    - Store in readonly field `_inner`

    The inner scorer must be accessible for CompositeScorer's cycle detection. Options:
    - Internal property: `internal IScorer Inner => _inner;` (preferred — keeps the field private, exposes read-only access within the assembly)
    - Or: make CompositeScorer's cycle detection use pattern matching with a well-known interface. Since both classes are in the same assembly, `internal` is cleanest.

    Score() method:
    ```
    1. var rawScore = _inner.Score(item, allItems)
    2. var min = double.MaxValue; var max = double.MinValue;
    3. for (var i = 0; i < allItems.Count; i++)
       {
           var s = _inner.Score(allItems[i], allItems);
           if (s < min) min = s;
           if (s > max) max = s;
       }
    4. if (max == min) return 0.5;
    5. return (rawScore - min) / (max - min);
    ```

    Use `max == min` for degenerate check (not `max - min < double.Epsilon`), since these are exact doubles from the same scorer. See RESEARCH.md P2.

    Zero-allocation: for-loop only, no LINQ/closures/boxing. The inner scorer calls are virtual dispatch (IScorer interface) but no heap allocation.

    After tests pass, update PublicAPI.Unshipped.txt with:


    Note: Do NOT expose `Inner` in PublicAPI — it is `internal`, not `public`.
