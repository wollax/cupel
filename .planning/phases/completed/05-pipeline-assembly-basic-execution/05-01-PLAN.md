  <implementation>
    GreedySlice is a sealed class in namespace `Wollax.Cupel` implementing ISlicer.

    Algorithm (O(N log N)):
    1. Early return: if scoredItems.Count == 0 or budget.TargetTokens <= 0, return []
    2. Build `(double Density, int Index)[]` array. For each item:
       - tokens = scoredItems[i].Item.Tokens
       - If tokens == 0: density = double.MaxValue (always include, free; negative-token items will never reach GreedySlice — they are filtered by Classify)
       - Else: density = scoredItems[i].Score / tokens
    3. Array.Sort with static delegate: descending by density, stable via ascending index
    4. Greedy fill loop: iterate sorted indices, add items whose tokens fit in remaining budget
       - Zero-token items: add immediately (cost 0)
       - Positive token items: add if tokens <= remainingTokens, then subtract
    5. Return selected items as `ContextItem[]` (not List<T>, to avoid extra allocation when possible; use a pre-allocated array and track count, then Array.Resize or use a List since exact count is unknown upfront -- List<ContextItem> is acceptable here per research note)

    See 05-RESEARCH.md "GreedySlice Algorithm" for complete code example.

    Zero-allocation discipline: for loops with indexer access, static comparison delegate. The List<ContextItem> and density array are per-execution allocations (acceptable).
  </implementation>

  <implementation>
    UShapedPlacer: sealed class in namespace `Wollax.Cupel` implementing IPlacer.

    Algorithm:
    1. Early return: if items.Count <= 1, extract and return items directly. Note: use `<= 1` (not `<= 2`). For 2 items, the full sort path produces the correct result (highest first at index 0, lowest second at index 1).
    2. Build `(double Score, int Index)[]`, sort descending by score (stable via index)
    3. Allocate result `ContextItem[]` of size items.Count
    4. Alternating placement: iterate sorted items. Even indices (i=0,2,4...) go to left pointer (starts at 0, increments). Odd indices (i=1,3,5...) go to right pointer (starts at Count-1, decrements).
    5. Return result array

    ChronologicalPlacer: sealed class in namespace `Wollax.Cupel` implementing IPlacer.

    Algorithm:
    1. Early return: if items.Count <= 1, extract and return items directly
    2. Build `(DateTimeOffset? Timestamp, int Index)[]`
    3. Array.Sort with static delegate: null timestamps sort to end, then ascending by timestamp, stable via index
    4. Build result ContextItem[] from sorted indices
    5. Return result array

    Both use for loops with indexer access and static comparison delegates per zero-allocation discipline.
