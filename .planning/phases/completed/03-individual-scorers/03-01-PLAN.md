    - HigherPriority_ScoresHigher: item with Priority=10 scores higher than Priority=1 (ordinal)
    - ThreeItems_LinearInterpolation: lowest → 0.0, middle → between, highest → 1.0 (ordinal)
    - TiedPriorities_ProduceEqualScores: same priority value → equal scores
    - MixedNullAndValid_NullGetsZero: null priorities get 0.0
    - AllNullPriorities_AllReturnZero: all null → all 0.0

    Use `List<ContextItem>` for allItems. Follow existing test style from ScoredItemTests (TUnit, async Task, Assert.That pattern).
  </behavior>
  <implementation>
    Both scorers are sealed classes in namespace Wollax.Cupel.Scoring implementing IScorer.

    RecencyScorer rank-counting algorithm:
    1. If item.Timestamp is null → return 0.0
    2. For-loop over allItems counting: (a) items with non-null Timestamp (countWithTimestamp), (b) items whose Timestamp is less than current item's Timestamp (rank)
    3. If countWithTimestamp <= 1 → return 1.0
    4. Return rank / (double)(countWithTimestamp - 1)

    PriorityScorer — identical pattern using item.Priority instead of item.Timestamp. Higher int value = higher rank.

    Zero-allocation rules: use `for (var i = 0; i < allItems.Count; i++)` with indexer access. No foreach, no LINQ, no closures.

    See RESEARCH.md Pattern 1 for exact code example.
  </implementation>
