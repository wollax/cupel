   | `Debugging()` | CUPEL007 | Priority(3), Kind(2), Recency(1) | Greedy | Chronological |

**Write tests first, then implement.**

1. Write `CupelOptionsTests` first (TDD), then implement `CupelOptions`:
   - Sealed class in `Wollax.Cupel` namespace
   - Internal field: `Dictionary<string, CupelPolicy>` with `StringComparer.OrdinalIgnoreCase`
   - Public API:
     - `AddPolicy(string intent, CupelPolicy policy)` — returns `CupelOptions` for chaining. Overwrites if intent already exists.
     - `GetPolicy(string intent)` — returns policy or throws `KeyNotFoundException`
     - `TryGetPolicy(string intent, [NotNullWhen(true)] out CupelPolicy? policy)` — returns bool
   - Validation:
     - `intent` must not be null or whitespace (`ArgumentException.ThrowIfNullOrWhiteSpace`)
     - `policy` must not be null (`ArgumentNullException.ThrowIfNull`)
   - `[JsonPropertyName]` not needed here — CupelOptions is a runtime registry, not serialized

2. Tests to write:
   - AddPolicy and GetPolicy round-trip
   - Case-insensitive key lookup (add "Chat", get "chat")
   - GetPolicy with unknown intent throws `KeyNotFoundException`
   - TryGetPolicy returns false for unknown intent
   - TryGetPolicy returns true and correct policy for known intent
   - AddPolicy with null/whitespace intent throws
   - AddPolicy with null policy throws
   - GetPolicy with null/whitespace intent throws
   - AddPolicy overwrites existing intent
   - Fluent chaining works (AddPolicy returns this)

3. Update `PublicAPI.Unshipped.txt` with all new public surface from this plan:

   - CupelOptions class, constructor, and all methods
