<objective>
Implement path-aware error handling for JSON deserialization. When CupelPolicy or ContextBudget constructor validation fails during deserialization, the error message should include the JSON path to the problem. This makes debugging large policy files practical.

STJ wraps constructor exceptions in JsonException during deserialization, and JsonException.Path provides the JSON path. The facade layer in CupelJsonSerializer should enhance error messages to include both the path and the invalid value, making errors actionable.
</objective>

  <implementation>
  Update `CupelJsonSerializer.Deserialize` (and DeserializeBudget) to enhance error handling:

  1. **Null/empty input guard:** Check for null/empty string before calling JsonSerializer.Deserialize. Throw `ArgumentNullException` for null, `JsonException` with clear message for empty.

  2. **Catch and enhance JsonException:** STJ wraps constructor validation exceptions (ArgumentException, ArgumentOutOfRangeException) in JsonException during deserialization. The JsonException.Path property provides the JSON path. The facade should:
     - Catch `JsonException` from deserialization
     - If it has an `InnerException` that's an `ArgumentException` or `ArgumentOutOfRangeException`, create a new exception with an enhanced message that includes `$.path: {original message}`
     - If it's a direct JsonException (malformed JSON, invalid enum), let it propagate with its existing message (which already includes path via JsonException.Path)

  3. **Null result guard:** If `JsonSerializer.Deserialize` returns null (e.g., JSON literal `null`), throw a `JsonException` with message `"Policy cannot be null."` or `"Budget cannot be null."`.

  4. **Exception type decision:** Use `JsonException` for all serialization-related errors (consistent, catchable with a single catch block). Wrap `ArgumentException`/`ArgumentOutOfRangeException` from constructors in `JsonException` with enhanced path info.

  Implementation pattern:
  ```csharp
  public static CupelPolicy Deserialize(string json, CupelJsonOptions? options = null)
  {
      ArgumentNullException.ThrowIfNull(json);
      if (json.Length == 0)
          throw new JsonException("JSON input cannot be empty.");

      try
      {
          var context = ResolveContext(options);
          return JsonSerializer.Deserialize(json, context.CupelPolicy)
              ?? throw new JsonException("Policy cannot be null. Received JSON literal 'null'.");
      }
      catch (JsonException ex) when (ex.InnerException is ArgumentException or ArgumentOutOfRangeException)
      {
          // Enhance with path context from the JsonException
          var path = ex.Path ?? "$";
          throw new JsonException($"{path}: {ex.InnerException.Message}", ex.InnerException);
      }
  }
  ```
  </implementation>
