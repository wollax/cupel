<objective>
Create the Wollax.Cupel.Json companion package with project scaffold, source-generated JsonSerializerContext, and basic CupelJsonSerializer facade. Prove round-trip serialization works for CupelPolicy and ContextBudget with full-fidelity data preservation.

This plan establishes the project infrastructure and core serialization path. The existing CupelPolicy model uses enums (ScorerType, SlicerType, PlacerType) with [JsonPropertyName] and [JsonConstructor] attributes. The source-generated context will serialize these directly — enums as camelCase strings, ContextKind via its existing JsonConverter, collections via constructor deserialization.

**Type discrimination strategy — enums instead of `$type` polymorphism:**

The CONTEXT.md mentions `$type` discriminators, but the domain model finalized in Phase 8 uses enums, NOT polymorphic interface hierarchies. STJ's `[JsonPolymorphic]`/`[JsonDerivedType]` attributes only apply to abstract/interface type hierarchies — they cannot be applied to enums or to classes that are already concrete. Since the model is enum-based, `$type` discriminators are inapplicable. Instead, enum string values serve as natural discriminators:

- **Scorers:** `ScorerEntry.Type` (ScorerType enum) serializes as `"type": "recency"` / `"priority"` / `"kind"` / `"tag"` / `"frequency"` / `"reflexive"`
- **Slicers:** `CupelPolicy.SlicerType` (SlicerType enum) serializes as `"slicerType": "greedy"` / `"knapsack"`
- **Placers:** `CupelPolicy.PlacerType` (PlacerType enum) serializes as `"placerType": "chronological"` / `"uShaped"`
- **Overflow:** `CupelPolicy.OverflowStrategy` (OverflowStrategy enum) serializes as `"overflowStrategy": "throw"` / `"truncate"` / `"proceed"`

This produces a JSON shape like `{"scorers": [{"type": "recency", "weight": 1.0}], "slicerType": "greedy", "placerType": "chronological"}` — functionally equivalent to `$type` discriminators but native to the domain model. No DTO mapping layer is needed.
</objective>

     - Mark `internal` — consumers use the CupelJsonSerializer facade, not the context directly
     - `UseStringEnumConverter = true` makes all enums serialize as camelCase strings
     - `PropertyNamingPolicy = CamelCase` is a safety net; existing `[JsonPropertyName]` attributes take precedence
     - `DefaultIgnoreCondition = WhenWritingNull` omits null optional properties (quotas, knapsackBucketSize, etc.)

     **Complete type inventory for source generation coverage:**
     The `[JsonSerializable(typeof(CupelPolicy))]` attribute causes STJ source gen to transitively discover and generate serializers for all types reachable from CupelPolicy's properties. The full graph is:
     - `CupelPolicy` — top-level: scorers, slicerType, placerType, overflowStrategy, quotas, name, description, deduplication, knapsackBucketSize
     - `ScorerEntry` — type (ScorerType enum), weight, kindWeights, tagWeights, innerScorers, scaleFactor
     - `ScorerType` enum — Recency, Priority, Kind, Tag, Frequency, Reflexive
     - `SlicerType` enum — Greedy, Knapsack
     - `PlacerType` enum — Chronological, UShaped
     - `OverflowStrategy` enum — Throw, Truncate, Proceed
     - `QuotaEntry` — kind (ContextKind), minPercent, maxPercent
     - `ContextKind` — uses existing `ContextKindJsonConverter` (string round-trip)
     - `IReadOnlyList<ScorerEntry>`, `IReadOnlyList<QuotaEntry>` — collection serialization via constructors
     - `IReadOnlyDictionary<ContextKind, double>` — kindWeights/tagWeights on ScorerEntry
     - `IReadOnlyDictionary<ContextKind, int>` — reservedSlots on ContextBudget (uses `ContextKindDictionaryConverter`)
     - `ContextBudget` — maxTokens, targetTokens, outputReserve, reservedSlots, estimationSafetyMarginPercent
