  <action>
  Create the `crates/cupel/examples/` directory and two example files.

  **basic_pipeline.rs** — Core pipeline flow tutorial:
  - Commented walkthrough style (every section has a `//` comment explaining what's happening)
  - Create 5-8 ContextItems with realistic LLM-domain content:
    - A system prompt (pinned)
    - 2-3 user messages with timestamps
    - 1-2 tool outputs
    - 1 document
  - Create a ContextBudget with realistic values (e.g., max_tokens=4096, target_tokens=3000)
  - Build a Pipeline with CompositeScorer (RecencyScorer + KindScorer::with_default_weights()), GreedySlice, ChronologicalPlacer
  - Run the pipeline and print results with println!
  - Print each selected item's content and kind
  - Use `fn main() -> Result<(), cupel::CupelError>` for clean error handling
  - Do NOT use asserts — examples are tutorials, not tests
  - Mix short content strings with descriptive names

  **serde_roundtrip.rs** — Serde feature tutorial:
  - Requires `--features serde` to compile — add `#![cfg(feature = "serde")]` at the top is NOT correct for examples. Instead, just use serde types directly and document in comments that `--features serde` is required.
  - Create a ContextItem and serialize to JSON with serde_json
  - Print the JSON
  - Deserialize back and print fields
  - Create a ContextBudget, serialize, then deserialize — demonstrating validation-on-deserialize
  - Show a failing deserialize case: invalid budget JSON (target > max) that produces an error, print the error
  - Use `fn main() -> Result<(), Box<dyn std::error::Error>>` since multiple error types
  - Commented walkthrough style
  - Note: serde_json is in dev-dependencies but NOT in regular dependencies. The example needs it. Add `serde_json = "1"` to `[dependencies]` as an optional dep gated on the `serde` feature, OR add it to `[dev-dependencies]` (it's already there). Since examples use dev-dependencies, serde_json is already available.
