<task type="auto">
  <name>Task 1: Verify crate name and create .gitignore</name>
  <files>.gitignore</files>
  <action>
  1. Verify crate name availability using the cargo CLI (authoritative check, not HTTP):
     ```bash
     cargo search cupel
     cargo info cupel 2>&1
     cargo search cupel-rs
     cargo info cupel-rs 2>&1
     ```
     Both should return "does not exist" or empty results. If either name is taken, note which is available. The preferred name is `cupel` (brainstorm recommendation). Fallback is `cupel-rs` (roadmap default).

  2. Create `.gitignore` at repo root. The repo currently has NO `.gitignore` — this must be created BEFORE any `cargo` commands that produce build artifacts.

     Contents:
     ```gitignore
     # Rust
     target/
     Cargo.lock

     # .NET
     **/bin/
     **/obj/
     TestResults/
     BenchmarkDotNet.Artifacts/

     # IDE
     .vs/
     *.user
     ```

     Rationale:
     - `target/` covers `crates/cupel/target/` and any future crate targets
     - `Cargo.lock` is NOT committed for library crates (Rust convention)
     - .NET patterns address the 20+ untracked directories visible in `git status`
     - IDE patterns prevent Visual Studio artifacts from being tracked
  </action>
  <verify>
  - `cargo search cupel` returns empty or "does not exist" (confirming name is available)
  - `.gitignore` exists at repo root and contains `target/` and `Cargo.lock` patterns
  - `cat .gitignore` shows both Rust and .NET sections
  </verify>
  <done>Crate name `cupel` (or fallback) confirmed available. `.gitignore` exists with Rust and .NET ignore patterns before any cargo commands run.</done>
</task>

<task type="auto">
  <name>Task 2: Create rust-toolchain.toml and extend .editorconfig</name>
  <files>rust-toolchain.toml, .editorconfig</files>
  <action>
  1. Create `rust-toolchain.toml` at repo root (parallels `global.json` for .NET SDK pinning):

     ```toml
     [toolchain]
     channel = "1.85.0"
     components = ["rustfmt", "clippy"]
     ```

     Use `channel = "1.85.0"` to pin MSRV directly. This satisfies the roadmap requirement ("pins Rust 2024 edition with MSRV 1.85") and ensures all contributors build with the exact minimum supported version. The `rust-version = "1.85"` field in Cargo.toml (Plan 02) mirrors this for consumer-facing MSRV declaration.

  2. Extend `.editorconfig` by appending Rust-specific sections AFTER the existing `[*.md]` section:

     ```ini
     [*.rs]
     indent_style = space
     indent_size = 4

     [*.toml]
     indent_style = space
     indent_size = 2
     ```

     The `[*.toml]` section covers both `Cargo.toml` and `rust-toolchain.toml`. No separate `[Cargo.toml]` section needed — same indent settings apply.
  </action>
  <verify>
  - `cat rust-toolchain.toml` shows `channel = "1.85.0"` with `rustfmt` and `clippy` components
  - `cat .editorconfig` shows `[*.rs]` section with `indent_size = 4` and `[*.toml]` section with `indent_size = 2`
  - `rustup show` (run from repo root) reflects the toolchain file is detected
  </verify>
  <done>`rust-toolchain.toml` exists at repo root with channel 1.85.0 (MSRV pin), rustfmt, and clippy. `.editorconfig` extended with `[*.rs]` (4-space indent) and `[*.toml]` (2-space indent) sections.</done>
</task>
