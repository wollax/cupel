  <action>
    1. Write `spec/src/conformance/levels.md`:
       - Define two conformance tiers:
         - **Required**: Core algorithm behavior. An implementation MUST pass all required vectors to claim Cupel conformance.
         - **Optional**: Edge cases and advanced features. An implementation MAY pass these for full conformance.
       - List which algorithms are in each tier:
         - Required: all scorers, GreedySlice, KnapsackSlice, ChronologicalPlacer, UShapedPlacer, end-to-end pipeline
         - Optional: QuotaSlice, deduplication edge cases, overflow handling, pinned item interactions

    2. Write `spec/src/conformance/format.md`:
       - Document the TOML test vector schema for each stage type:
         - **Scoring vectors**: `[test]` (name, stage="scoring", scorer), `[[items]]` (input items), `[[expected]]` (content + score_approx), `[tolerance]` (score_epsilon)
         - **Slicing vectors**: `[test]` (name, stage="slicing", slicer), `[budget]` (target_tokens), `[[scored_items]]` (pre-scored input), `[expected]` (selected_contents list)
         - **Placing vectors**: `[test]` (name, stage="placing", placer), `[[items]]` (items with scores), `[expected]` (ordered_contents list)
         - **Pipeline vectors**: `[test]` (name, stage="pipeline"), `[budget]`, `[config]` (slicer, placer, deduplication, scorers), `[[items]]`, `[[expected_output]]` (ordered contents)
       - Document field types: content=string, tokens=integer, score=float64, timestamp=RFC 3339 datetime, kind=string, tags=array of strings, etc.
       - Address P2: score_approx fields use epsilon tolerance (default 1e-9). Pipeline/slicer/placer vectors compare item sets and ordering, not scores.

    3. Write `spec/src/conformance/running.md`:
       - Explain the general approach: parse TOML, construct inputs, run algorithm, compare outputs
       - Provide pseudocode for a conformance test runner
       - List TOML parsing libraries for common languages (Rust: `toml`, C#: `Tomlyn`, Python: `tomllib`, Go: `pelletier/go-toml`)
       - Explain score comparison: `abs(actual - expected) < epsilon`
       - Explain set comparison for slicer output (order may not match) vs ordered comparison for placer/pipeline output

    4. Write `spec/src/changelog.md`:
       - Version 1.0.0 (current date): Initial specification release
       - Covers: data model, 6 pipeline stages, 8 scorers, 4 slicers (3 sync + StreamSlice noted), 2 placers
       - Conformance suite: N required vectors, M optional vectors (fill in actual counts)

    5. Write `conformance/README.md`:
       - Brief explanation of directory structure
       - How to use the vectors
       - Link to the rendered spec on GitHub Pages

    6. Create **required** test vectors (TOML files):

       **Scoring** (`conformance/required/scoring/`):
       - `recency-basic.toml`: 3 items with timestamps, verify rank ordering
       - `recency-null-timestamps.toml`: mix of null and non-null timestamps
       - `priority-basic.toml`: 3 items with different priorities
       - `priority-null.toml`: mix of null and non-null priorities
       - `kind-default-weights.toml`: one item per default kind, verify weight lookup
       - `kind-unknown.toml`: item with unknown kind, verify 0.0
       - `tag-basic.toml`: items with various tag overlaps
       - `tag-no-tags.toml`: item with no tags, verify 0.0
       - `frequency-basic.toml`: items sharing tags, verify proportion calculation
       - `reflexive-basic.toml`: items with futureRelevanceHint values
       - `reflexive-null.toml`: null hint, verify 0.0
       - `composite-weighted.toml`: two child scorers with weights, verify weighted average
       - `scaled-basic.toml`: inner scorer with varying outputs, verify min-max normalization

       **Slicing** (`conformance/required/slicing/`):
       - `greedy-density.toml`: items with different score/token ratios (use example from research)
       - `greedy-zero-tokens.toml`: zero-token items always included
       - `greedy-exact-fit.toml`: items that exactly fill budget
       - `knapsack-basic.toml`: case where knapsack selects differently from greedy
       - `knapsack-zero-tokens.toml`: zero-token items pre-filtered

       **Placing** (`conformance/required/placing/`):
       - `chronological-basic.toml`: items with timestamps, verify ascending order
       - `chronological-null-timestamps.toml`: null timestamps sort to end
       - `u-shaped-basic.toml`: items with scores, verify even/odd edge placement
       - `u-shaped-equal-scores.toml`: equal scores, verify stable tiebreaking

       **Pipeline** (`conformance/required/pipeline/`):
       - `greedy-chronological.toml`: basic end-to-end with greedy + chronological
       - `greedy-ushaped.toml`: basic end-to-end with greedy + u-shaped
       - `knapsack-chronological.toml`: end-to-end with knapsack + chronological
       - `pinned-items.toml`: pinned items bypass scoring, appear in output

    7. Create **optional** test vectors (TOML files):

       **Scoring** (`conformance/optional/scoring/`):
       - `recency-single-item.toml`: single timestamped item → 1.0
       - `recency-all-null.toml`: all null timestamps → all 0.0
       - `scaled-degenerate.toml`: all equal inner scores → 0.5
       - `composite-nested.toml`: composite containing composite

       **Slicing** (`conformance/optional/slicing/`):

       **Pipeline** (`conformance/optional/pipeline/`):

       - `all-pinned.toml`: all items pinned → all appear, placer orders
