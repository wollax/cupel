<task type="auto">
  <name>Task 2: Add conformance drift guard step</name>
  <files>.github/workflows/ci-rust.yml</files>
  <action>
    Insert a new step named "Conformance drift guard" in the `check` job, placed after the "Cache" step and before the "Format" step. This positions the drift guard before any Rust compilation steps, catching drift early without waiting for build/test.

    The step content:

    ```yaml
      - name: Conformance drift guard
        run: |
          if [ -d spec/conformance/required ] && [ -d crates/cupel/conformance/required ]; then
            diff -rq spec/conformance/required crates/cupel/conformance/required \
              || { echo "ERROR: Conformance drift detected in required/ — spec and crates copies have diverged. Run: diff -r spec/conformance/required crates/cupel/conformance/required"; exit 1; }
          fi
          if [ -d spec/conformance/optional ] && [ -d crates/cupel/conformance/optional ]; then
            diff -rq spec/conformance/optional crates/cupel/conformance/optional \
              || { echo "ERROR: Conformance drift detected in optional/ — spec and crates copies have diverged. Run: diff -r spec/conformance/optional crates/cupel/conformance/optional"; exit 1; }
          fi
          echo "Conformance vectors in sync."
    ```

    Key requirements:
    - Both `required/` and `optional/` pairs are checked
    - Each pair has a `-d` existence guard (optional/ does not yet exist)
    - Error messages include a remediation hint (the `diff -r` command to run locally)
    - The step uses `exit 1` on divergence to fail the CI job
  </action>
  <verify>
    Verify the step appears in ci-rust.yml between Cache and Format steps. Verify both required/ and optional/ directory pairs are checked. Verify `-d` existence guards are present for both pairs. Verify `.github/workflows/ci-rust.yml` still remains in the paths: arrays (no accidental removal).
  </verify>
  <done>CI drift guard step exists in ci-rust.yml, positioned before compilation steps, checking both required/ and optional/ conformance directories with existence guards.</done>
</task>
