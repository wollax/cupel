    Create a new class library project `Wollax.Cupel.Extensions.DependencyInjection`:

    **Project file** — follow the existing pattern from `Wollax.Cupel.Json.csproj`:
    - `IsPackable` true
    - `Description` for NuGet
    - `ProjectReference` to `Wollax.Cupel`
    - `PackageReference` to `Microsoft.CodeAnalysis.PublicApiAnalyzers` (PrivateAssets=All)
    - `PackageReference` to `Microsoft.Extensions.DependencyInjection.Abstractions`
    - `PackageReference` to `Microsoft.Extensions.Options`
    - Empty `PublicAPI.Shipped.txt` (with `#nullable enable`) and `PublicAPI.Unshipped.txt`
    - `InternalsVisibleTo` for the test project

    **Directory.Packages.props** — add:
    - `Microsoft.Extensions.DependencyInjection.Abstractions` Version="10.0.0"
    - `Microsoft.Extensions.Options` Version="10.0.0"
    - `Microsoft.Extensions.DependencyInjection` Version="10.0.0" (needed by test project for concrete container)

    **Cupel.slnx** — add the new project to `/src/` folder and test project to `/tests/` folder.

    **CupelServiceCollectionExtensions.cs** — static class with extension methods:

    1. `AddCupel(this IServiceCollection services, Action<CupelOptions> configure)`:
       - Calls `services.Configure(configure)` to register `IOptions<CupelOptions>`
       - Returns `IServiceCollection` for chaining

    2. `AddCupelPipeline(this IServiceCollection services, string intent, ContextBudget budget)`:
       - The `budget` parameter is provided at registration time intentionally — budget is per-pipeline configuration (token limits vary by model/deployment), not part of `CupelOptions`. Each named pipeline gets its own budget.
       - Validates `intent` is not null/whitespace, `budget` is not null
       - Registers a keyed transient `CupelPipeline` using `services.AddKeyedTransient<CupelPipeline>(intent, factory)`
       - Factory resolves `IOptions<CupelOptions>`, calls `GetPolicy(intent)`, builds pipeline via `CupelPipeline.CreateBuilder().WithPolicy(policy).WithBudget(budget).Build()`
       - Returns `IServiceCollection` for chaining

    3. Optionally: `AddCupelTracing(this IServiceCollection services)` to register `DiagnosticTraceCollector` as transient `ITraceCollector`

    Use XML doc comments on all public members. Namespace: `Microsoft.Extensions.DependencyInjection` (standard .NET convention for extension methods on IServiceCollection).

    After creating the source files, run `dotnet build` to populate `PublicAPI.Unshipped.txt` via the analyzer, then copy the generated entries into the file.
