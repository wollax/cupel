       - Define pub trait Slicer: `fn slice(&self, sorted: &[ScoredItem], budget: &ContextBudget) -> Vec<ContextItem>;`
       - Re-export all slicer types

    2. Create `src/slicer/greedy.rs` — GreedySlice:
       - Unit struct implementing Slicer
       - Algorithm per spec GREEDY-SLICE:
         a. If empty or targetTokens <= 0, return empty
         b. Compute value densities: tokens=0 → f64::MAX, else score/tokens
         c. Stable sort by (density desc, index asc) using f64::total_cmp()
         d. Greedy fill: zero-token items always included, others included if tokens <= remaining
       - Output order is density-sorted order (not original order — placer handles final order)

    3. Create `src/slicer/knapsack.rs` — KnapsackSlice:
       - Constructed with bucket_size (default 100, must be > 0)
       - Algorithm per spec KNAPSACK-SLICE:
         a. Pre-filter zero-token items (always included)
         b. Score scaling: max(0, floor(score * 10000.0)) as i64
         c. Capacity: floor(targetTokens / bucketSize) — if 0, return only zero-token items
         d. Weight discretization: ceil(tokens / bucketSize) — use (tokens + bucket_size - 1) / bucket_size
         e. 1D DP array + 2D boolean keep table
         f. DP: for each item i, for w from capacity downto dw, if dp[w-dw]+dv > dp[w] then update
         g. Reconstruct: iterate i from n-1 downto 0
         h. Result: zero-token items first, then DP-selected

    4. Create `src/slicer/quota.rs` — QuotaSlice:
       - QuotaEntry: kind (ContextKind), require (f64 percent), cap (f64 percent)
       - QuotaSlice constructed with Vec<QuotaEntry> and inner slicer (Box<dyn Slicer>)
       - Validate: require <= cap for each, sum of requires <= 100%
       - Algorithm per spec QUOTA-SLICE:
         a. Partition items by ContextKind (case-insensitive — use HashMap<ContextKind, Vec<ScoredItem>>)
         b. Compute candidate token mass per kind
         c. Distribute budget: requireTokens = floor(require/100 * target), capTokens = floor(cap/100 * target)
         d. Unassigned = target - totalRequired. Proportional distribution by candidate mass.
         e. Per-kind: budget = require + proportional, clamped to cap
         f. Delegate per-kind selection to inner slicer with sub-budget

         b. Allocate result array of length N
         c. Even ranks (0, 2, 4...) fill from left pointer

    4. Create `src/pipeline/classify.rs`:

       - Exclude items with tokens < 0
       - tokens < 0 check BEFORE pinned check
       - Validate pinned budget: sum(pinned.tokens) <= maxTokens - outputReserve

    5. Create `src/pipeline/score.rs`:
       - `pub(crate) fn score_items(scoreable: &[ContextItem], scorer: &dyn Scorer) -> Vec<ScoredItem>`

    6. Create `src/pipeline/deduplicate.rs`:
       - `pub(crate) fn deduplicate(scored: Vec<ScoredItem>, enabled: bool) -> Vec<ScoredItem>`
       - If not enabled, return as-is

       - Collect survivors in original order

    7. Create `src/pipeline/sort.rs`:
       - `pub(crate) fn sort_scored(deduped: Vec<ScoredItem>) -> Vec<ScoredItem>`

    8. Create `src/pipeline/slice.rs`:
       - `pub(crate) fn compute_effective_budget(budget: &ContextBudget, pinned_tokens: i64) -> ContextBudget`
       - effectiveMax = max(0, maxTokens - outputReserve - pinnedTokens)
       - effectiveTarget = min(max(0, targetTokens - pinnedTokens), effectiveMax)
       - Create new ContextBudget with only maxTokens=effectiveMax, targetTokens=effectiveTarget
       - `pub(crate) fn slice_items(sorted: &[ScoredItem], budget: &ContextBudget, pinned_tokens: i64, slicer: &dyn Slicer) -> Vec<ContextItem>`

    9. Create `src/pipeline/place.rs`:
       - `pub(crate) fn place_items(pinned: &[ContextItem], sliced: &[ScoredItem], budget: &ContextBudget, overflow_strategy: OverflowStrategy, placer: &dyn Placer) -> Result<Vec<ContextItem>, CupelError>`
       - Step 1: Merge — pinned items as ScoredItem with score=1.0, then sliced items
       - Step 2: Overflow detection — sum merged tokens, compare against budget.target_tokens (ORIGINAL, not effective)
         - Throw: return Err(CupelError::Overflow)
         - Truncate: keep pinned always, keep non-pinned while tokens fit
         - Proceed: no error (observer notification is implementation-optional, skip for v1.0)
       - Step 3: Delegate to placer for final ordering
       - IMPORTANT: The sliced items passed to this function need their scores. The pipeline must re-associate scores after slicing. The slicer returns Vec<ContextItem> (without scores). The pipeline must match sliced items back to their scored counterparts. Use content matching or track indices.
