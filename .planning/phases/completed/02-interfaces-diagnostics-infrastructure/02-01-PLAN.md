<objective>
Build the tracing infrastructure and supporting value types that every pipeline stage depends on.

Purpose: ITraceCollector is passed to every pipeline stage (ISlicer, IPlacer). NullTraceCollector enables zero-allocation disabled paths. DiagnosticTraceCollector enables debugging and testing. ScoredItem is the inter-stage data type between scorer, slicer, and placer. These must exist before pipeline interfaces can be defined.

Output: ITraceCollector interface, two implementations (NullTraceCollector, DiagnosticTraceCollector), supporting enums (PipelineStage, TraceDetailLevel, ExclusionReason), TraceEvent readonly record struct, ScoredItem readonly record struct — all with comprehensive TDD tests.
</objective>
