  <name>Task 1: Create crates.io API token and trigger first publish</name>
  <action>
The first publish requires manual steps that cannot be automated:

1. **Create crates.io API token:**
   - Go to https://crates.io/settings/tokens/new
   - Name: `cupel-first-publish` (or similar)
   - Scopes: `publish-new` + `publish-update`
   - Crate scope: `cupel`
   - Copy the token

2. **Add token as GitHub repository secret:**
   - Go to https://github.com/wollax/cupel/settings/secrets/actions
   - Create secret: `CARGO_REGISTRY_TOKEN` with the token value

3. **Merge Plan 01 changes to main** (workflow fixes must be on main before triggering)

4. **Trigger the release workflow:**
   - Go to https://github.com/wollax/cupel/actions/workflows/release-rust.yml
   - Click "Run workflow" from `main` branch
   - Leave dry-run unchecked (this is the real publish)
   - Wait for workflow to complete successfully

5. **Verify crate is live:**
   - Visit https://crates.io/crates/cupel
   - Confirm version 1.0.0 is published
   - Confirm README is rendered
   - Confirm categories and keywords are visible
   - Check https://docs.rs/cupel — docs.rs build should be triggered automatically

6. **Verify GitHub Release:**
   - Visit https://github.com/wollax/cupel/releases
   - Confirm `rust-v1.0.0` tag and release exist
   - Confirm release is marked as "Latest"

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Configure OIDC trusted publishing on crates.io</name>
  <action>
Now that the crate exists on crates.io, configure trusted publishing:

1. **Add trusted publisher on crates.io:**
   - Go to https://crates.io/crates/cupel/settings (or crate settings page)
   - Navigate to "Trusted Publishers" section
   - Add a GitHub Actions publisher with:
     - Owner: `wollax`
     - Repository: `cupel`
     - Workflow filename: `release-rust.yml`
     - Environment: `release`

2. **Delete the API token:**
   - Go to https://github.com/wollax/cupel/settings/secrets/actions
   - Delete the `CARGO_REGISTRY_TOKEN` repository secret
   - Go to https://crates.io/settings/tokens
   - Revoke/delete the `cupel-first-publish` token

OIDC verification will happen on the next natural version bump (e.g., when Phase 20 publishes 1.1.0 with serde feature). The workflow already has the OIDC auth action from Plan 01.
  </action>
  <resume-signal>Type "oidc-configured" when trusted publishing is set up and the token is deleted, or describe any issues.</resume-signal>
</task>
