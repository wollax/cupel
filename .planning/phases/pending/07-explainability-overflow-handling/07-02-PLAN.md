  RED: Write tests for report population when DiagnosticTraceCollector is used.

  Tests to add (in existing CupelPipelineTests.cs or a new section):
  - Execute with DiagnosticTraceCollector → Report.Included contains selected items with correct scores and InclusionReason.Scored
  - Execute with pinned items → Report.Included contains pinned items with InclusionReason.Pinned and score 1.0
  - Execute where items excluded by slicer → Report.Excluded contains them with ExclusionReason.BudgetExceeded
  - Execute with deduplication → Report.Excluded contains duplicates with ExclusionReason.Deduplicated and DeduplicatedAgainst set
  - Execute with negative-token items → Report.Excluded contains them with ExclusionReason.NegativeTokens
  - Report.TotalCandidates = total input item count
  - Report.TotalTokensConsidered = sum of all non-negative token items
  - Report.Excluded is ordered by score descending
  - Execute without DiagnosticTraceCollector → Report is null (existing behavior preserved)

  GREEN: Refactor CupelPipeline.

  1. **Add fields** to CupelPipeline: `_overflowStrategy` (OverflowStrategy), `_overflowObserver` (Action<OverflowEvent>?). Update internal constructor.

  2. **Extract `ExecuteCore()`** — private method containing the full pipeline logic. Both `Execute()` and `DryRun()` call it.
     Signature: `private ContextResult ExecuteCore(IReadOnlyList<ContextItem> items, ITraceCollector trace, bool isDryRun)`
     `Execute()` becomes a thin wrapper that calls `ExecuteCore(items, trace, isDryRun: false)`.

  3. **Wire ReportBuilder** — Inside ExecuteCore, after the `traceCollector is DiagnosticTraceCollector` check:
     ```csharp
     ReportBuilder? reportBuilder = trace is DiagnosticTraceCollector ? new ReportBuilder() : null;
     ```
     Then at each pipeline stage, populate the report builder:

     a. **CLASSIFY stage** — After classify loop:
        - `reportBuilder?.SetTotalCandidates(items.Count)`
        - For negative-token items: `reportBuilder?.AddExcluded(item, 0.0, ExclusionReason.NegativeTokens)`
        - Compute TotalTokensConsidered (sum of non-negative token items)

     b. **DEDUPLICATE stage** — When dedup removes items:
        - Track which item each duplicate was deduplicated against (the surviving item with the same content)
        - `reportBuilder?.AddExcluded(item, score, ExclusionReason.Deduplicated, deduplicatedAgainst: survivingItem)`

     c. **SLICE stage** — Diff-based detection. After slicing, compare sorted array to slicedSet:
        - Items in sorted but NOT in slicedSet → `reportBuilder?.AddExcluded(item, score, ExclusionReason.BudgetExceeded)`

     d. **MERGE stage** — After merging pinned + sliced:
        - Pinned items: `reportBuilder?.AddIncluded(item, 1.0, InclusionReason.Pinned)`
        - Sliced items: `reportBuilder?.AddIncluded(item, score, InclusionReason.Scored)`
        - Zero-token items that made it through: check if tokens == 0 → `InclusionReason.ZeroToken`

  4. **Build report** — Replace existing report construction:
     ```csharp
     SelectionReport? report = null;
     if (trace is DiagnosticTraceCollector diagnosticCollector && reportBuilder is not null)
     {
         report = reportBuilder.Build(diagnosticCollector.Events);
     }
     ```

  REFACTOR: Ensure no behavior change for non-diagnostic paths. The `if (sw is not null)` pattern remains.

  <action>
  RED: Write tests first.

  **OverflowStrategy tests** (new file `OverflowStrategyTests.cs`):
  - Default strategy is Throw (pipeline built without WithOverflowStrategy)
  - Throw: when sliced + pinned tokens > TargetTokens, throws `OverflowException` with message containing token counts
  - Truncate: when overflow occurs, removes lowest-scored sliced items until within budget. Removed items get ExclusionReason.PinnedOverride in report.
  - Proceed: when overflow occurs, callback `Action<OverflowEvent>` is invoked with correct TokensOverBudget, OverflowingItems, Budget
  - Proceed without callback: no exception, items pass through
  - Pinned items alone > MaxTokens: still throws InvalidOperationException (unchanged)
  - No overflow: all strategies behave identically (no exception, no callback)

  **DryRun tests** (new file `DryRunTests.cs`):
  - DryRun() returns ContextResult with Report always populated (even without passing a traceCollector)
  - DryRun() result matches Execute() result for same input (same items, same order)
  - DryRun() is idempotent: two calls with same items → identical results
  - DryRun() Report contains Included and Excluded items
  - DryRun() with pinned items works correctly
  - DryRun() does not accept traceCollector parameter (it creates its own internally)

  GREEN: Implement.

  1. **PipelineBuilder changes:**
     - Add fields: `_overflowStrategy` (OverflowStrategy, default `Throw`), `_overflowObserver` (Action<OverflowEvent>?)
     - Add method:
       ```csharp
       public PipelineBuilder WithOverflowStrategy(OverflowStrategy strategy, Action<OverflowEvent>? onOverflow = null)
       ```
       Validates strategy is a defined enum value. Stores both fields.
     - Pass `_overflowStrategy` and `_overflowObserver` to CupelPipeline constructor.

  2. **CupelPipeline overflow handling** — In ExecuteCore(), after MERGE and before PLACE:
     ```
     var mergedTokens = sum of merged[i].Item.Tokens
     if mergedTokens > _budget.TargetTokens:
       switch _overflowStrategy:
         Throw → throw new OverflowException(...)
         Truncate → remove lowest-scored sliced items until fits, mark removed as PinnedOverride
         Proceed → invoke _overflowObserver with OverflowEvent, continue
     ```
     For Truncate: iterate merged from end (lowest score) and remove items that are NOT pinned until within budget. Track removed items as ExclusionReason.PinnedOverride in reportBuilder.

  3. **DryRun() method** on CupelPipeline:
     ```csharp
     public ContextResult DryRun(IReadOnlyList<ContextItem> items)
     {
         ArgumentNullException.ThrowIfNull(items);
         var trace = new DiagnosticTraceCollector();
         return ExecuteCore(items, trace, isDryRun: true);
     }
     ```
     The `isDryRun` parameter is currently unused but reserved for future side-effect gating.

  REFACTOR: Ensure OverflowException message is descriptive. XML doc comments on DryRun() and WithOverflowStrategy().
