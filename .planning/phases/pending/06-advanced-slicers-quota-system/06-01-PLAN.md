<objective>
Implement KnapsackSlice — a 0/1 knapsack DP slicer with configurable bucket discretization that produces provably optimal selections within a token budget.

Purpose: KnapsackSlice is the optimization slicer that differentiates Cupel from naive greedy truncation. Where GreedySlice uses a heuristic (value density), KnapsackSlice finds the mathematically optimal item subset for a given budget. The bucket discretization trades precision for performance at scale.

Output: KnapsackSlice sealed class implementing ISlicer, with full TDD test coverage.
</objective>

  <implementation>
    KnapsackSlice is a sealed class in namespace `Wollax.Cupel` implementing ISlicer.

    Constructor: `KnapsackSlice(int bucketSize = 100)`. Validate bucketSize > 0.

    Algorithm (O(n * W/B) time, O(W/B) space):

    1. Early return: if scoredItems.Count == 0 or budget.TargetTokens <= 0, return []

    2. Pre-filter zero-token items: iterate scoredItems, separate into zeroTokenItems (List<ContextItem>) and candidates (parallel arrays: int[] weights, int[] values, ContextItem[] items). For candidates:
       - weight = item.Tokens (raw, not yet discretized)
       - value = (int)(score * 10000) — int-scaled for DP correctness
       - Skip items with Tokens <= 0 that aren't zero-token (shouldn't happen given pipeline filtering, but defensive)

    3. Discretize:
       - capacity = budget.TargetTokens / bucketSize (floor division — conservative)
       - If capacity == 0 and there are candidates with tokens > 0, return zeroTokenItems only
       - discretizedWeights[i] = (weights[i] + bucketSize - 1) / bucketSize (ceiling division — overestimates item cost for feasibility)

    4. Rent DP array from ArrayPool<int>.Shared, size = capacity + 1. Zero it. Wrap usage in try/finally for return.

    5. Fill DP table (1D, reverse iteration):
       ```
       for i = 0 to candidates.Length - 1:
           for w = capacity down to discretizedWeights[i]:
               dp[w] = Math.Max(dp[w], dp[w - discretizedWeights[i]] + values[i])
       ```
       CRITICAL: inner loop MUST iterate in REVERSE. Forward iteration = unbounded knapsack (wrong).

    6. Reconstruct solution (reverse scan):
       ```
       w = capacity
       for i = candidates.Length - 1 down to 0:
           if w >= discretizedWeights[i] && dp[w] != dp[w - discretizedWeights[i]]:
               select candidate i
               w -= discretizedWeights[i]
       ```

    7. Build result: combine zeroTokenItems + selected candidates. Return as IReadOnlyList<ContextItem>.

    8. Return DP array to ArrayPool in finally block. Zero the array before return (clearArray: true).

    Zero-allocation discipline: for loops with indexer, no LINQ. The ArrayPool rental is the key optimization — avoids GC pressure for the DP table.
  </implementation>
