<objective>
Implement IAsyncSlicer interface and StreamSlice — an online/streaming slicer that processes IAsyncEnumerable sources without materializing the full collection.

Purpose: StreamSlice handles unbounded or large streaming sources where materializing all items before slicing is impractical. It processes items in configurable micro-batches and stops consuming when the budget is full.

Output: IAsyncSlicer interface and StreamSlice sealed class with full TDD test coverage. SLICE-05 requirement.
</objective>

    StreamSlice (namespace Wollax.Cupel.Slicing):
    - Sealed class implementing IAsyncSlicer
    - Constructor: `StreamSlice(int batchSize = 32)` — validates batchSize > 0
    - Field: `private readonly int _batchSize;`

    Algorithm (online greedy with micro-batching):
    1. Early return: if budget.TargetTokens <= 0, return []
    2. Create a linked CancellationTokenSource from the incoming token (for budget-full cancellation)
    3. Initialize: `var selected = new List<ContextItem>(); var remainingTokens = budget.TargetTokens;`
    4. Consume in micro-batches:
       ```csharp
       var batch = new List<ScoredItem>(_batchSize);
       await foreach (var item in scoredItems.WithCancellation(cts.Token).ConfigureAwait(false))
       {
           batch.Add(item);
           if (batch.Count >= _batchSize)
           {
               ProcessBatch(batch, ref remainingTokens, selected);
               batch.Clear();
               if (remainingTokens <= 0)
               {
                   cts.Cancel(); // Signal upstream to stop
                   break;
               }
           }
       }
       // Process final partial batch
       if (batch.Count > 0 && remainingTokens > 0)
       {
           ProcessBatch(batch, ref remainingTokens, selected);
       }
       ```
    5. ProcessBatch: sort batch by score descending (within batch), then greedy fill:
       ```csharp
       private static void ProcessBatch(List<ScoredItem> batch, ref int remainingTokens, List<ContextItem> selected)
       {
           // Sort batch by score descending for within-batch prioritization
           batch.Sort(static (a, b) => b.Score.CompareTo(a.Score));
           for (var i = 0; i < batch.Count; i++)
           {
               var item = batch[i].Item;
               if (item.Tokens == 0)
               {
                   selected.Add(item);
               }
               else if (item.Tokens <= remainingTokens)
               {
                   selected.Add(item);
                   remainingTokens -= item.Tokens;
               }
           }
       }
       ```
    6. Dispose the CancellationTokenSource.
    7. Return selected.

    Key details:
    - ConfigureAwait(false) on all awaits
    - WithCancellation() on the IAsyncEnumerable
    - CancellationTokenSource.CreateLinkedTokenSource for combining external cancellation with budget-full
    - try/finally for CTS disposal
    - OperationCanceledException from the source is caught and swallowed when WE initiated the cancellation (budget full). Re-throw if external cancellation.
