  <implementation>
    CompositeScorer is a sealed class in namespace Wollax.Cupel.Scoring implementing IScorer.

    Constructor signature: `public CompositeScorer(IReadOnlyList<(IScorer Scorer, double Weight)> entries)`

    Constructor logic:
    1. ArgumentNullException.ThrowIfNull(entries)
    2. if (entries.Count == 0) throw new ArgumentException("At least one scorer entry is required.", nameof(entries))
    3. For-loop over entries:
       - ArgumentNullException.ThrowIfNull(scorer, $"entries[{i}].Scorer")
       - ArgumentOutOfRangeException.ThrowIfNegativeOrZero(weight, $"entries[{i}].Weight")
       - if (!double.IsFinite(weight)) throw ArgumentOutOfRangeException
       - Accumulate totalWeight, populate IScorer[] and double[] arrays
    4. Normalize weights: weights[i] /= totalWeight
    5. Run cycle detection: static DFS method with HashSet<IScorer>(ReferenceEqualityComparer.Instance)
    6. Assign to readonly fields _scorers and _normalizedWeights

    Score() method: simple multiply-accumulate for-loop returning weighted sum.

    Cycle detection: private static void DetectCycles(IScorer root, IScorer[] children)
    - Uses visited HashSet and inPath HashSet (both with ReferenceEqualityComparer.Instance)
    - Traverses CompositeScorer children recursively
    - Pattern-matches on ScaledScorer to traverse its inner scorer (prepare for Plan 04-02 — initially this branch won't match anything since ScaledScorer doesn't exist yet, but the code is ready)
    - Throws ArgumentException on cycle detection

    Internal access: The cycle detection needs to access child scorers of CompositeScorer instances found during traversal. Store _scorers as `internal` (or use a pattern where DetectCycles accesses it via the same class). Since DetectCycles is a private static method in CompositeScorer, it can access private fields of other CompositeScorer instances directly.

    See RESEARCH.md patterns 1 and 2 for exact code examples.
