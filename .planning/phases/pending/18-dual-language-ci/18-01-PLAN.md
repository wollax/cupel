  <name>Task 1: Create cargo-deny configuration and Rust CI workflow</name>
  <files>
    crates/cupel/deny.toml
    .github/workflows/ci-rust.yml
  </files>
  <action>
**Create `crates/cupel/deny.toml`** with the following policy:
- `[advisories]`: version 2, `yanked = "deny"`
- `[licenses]`: version 2, allow list appropriate for permissive OSS: MIT, Apache-2.0, Apache-2.0 WITH LLVM-exception, BSD-2-Clause, BSD-3-Clause, ISC, Unicode-3.0, Unicode-DFS-2016, Zlib. Set `confidence-threshold = 0.93`
- `[bans]`: `multiple-versions = "warn"`, `wildcards = "deny"`, `highlight = "simplest-path"`
- `[sources]`: `unknown-registry = "deny"`, `unknown-git = "deny"`, `allow-registry = ["https://github.com/rust-lang/crates.io-index"]`

**Create `.github/workflows/ci-rust.yml`**:
- Name: `CI (Rust)`
- Triggers: `push` to `main` and `pull_request` to `main`, both with `paths` filter: `crates/**`, `rust-toolchain.toml`, `.github/workflows/ci-rust.yml`
- Note: Root `conformance/**` is NOT listed because the Rust crate vendors its own conformance vectors at `crates/cupel/conformance/` (Phase 17). The `crates/**` glob covers vendored vectors. Root `conformance/` changes only affect `spec.yml`.
- Single job `check` on `ubuntu-latest`
- Steps:
  1. `actions/checkout@v4`
  2. `dtolnay/rust-toolchain@1.85.0` with components: `rustfmt, clippy` (pin to MSRV, not "stable")
  3. `Swatinem/rust-cache@v2` with `workspaces: "crates/cupel -> target"`
  4. `cargo fmt --check --manifest-path crates/cupel/Cargo.toml`
  5. `cargo clippy --manifest-path crates/cupel/Cargo.toml -- -D warnings`
  6. `cargo test --manifest-path crates/cupel/Cargo.toml`
  7. `EmbarkStudios/cargo-deny-action@v2` with `manifest-path: crates/cupel/Cargo.toml`

Match the style conventions of the existing `ci.yml` (step naming, indentation).
  </action>
  <verify>
Run locally:
```bash
cd crates/cupel && cargo fmt --check && cargo clippy -- -D warnings && cargo test
cargo install cargo-deny --locked 2>/dev/null || true
cargo deny --manifest-path crates/cupel/Cargo.toml check
```
Validate YAML syntax of ci-rust.yml (use `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci-rust.yml'))"` or similar).
  </verify>
