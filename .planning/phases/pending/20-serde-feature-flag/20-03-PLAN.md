<name>Task 1: Write serde test suite</name>
<files>crates/cupel/tests/serde.rs</files>
<action>
Create `crates/cupel/tests/serde.rs` — an integration test file gated behind `#[cfg(feature = "serde")]`.

The entire file should be wrapped in a `#[cfg(feature = "serde")]` module or use a top-level attribute. Use `serde_json` for serialization (already in dev-dependencies).

**Test categories to implement:**

1. **Roundtrip tests** (serialize → deserialize, assert equality):
   - `roundtrip_context_kind` — create via `ContextKind::new("Document")`, serialize, deserialize, assert `as_str()` matches
   - `roundtrip_context_source` — create via `ContextSource::new("Tool")`, serialize, deserialize, assert `as_str()` matches
   - `roundtrip_overflow_strategy` — test all 3 variants (Throw, Truncate, Proceed)
   - `roundtrip_context_item_minimal` — only required fields (content + tokens), verify defaults survive roundtrip
   - `roundtrip_context_item_full` — all fields populated including timestamp, metadata, tags
   - `roundtrip_context_budget` — valid budget with reserved_slots HashMap containing ContextKind keys
   - `roundtrip_quota_entry` — valid entry with kind, require, cap
   - `roundtrip_scored_item` — item + score

2. **Validation rejection tests** (deserialize invalid JSON, assert error):
   - `reject_empty_context_kind` — `""` should fail
   - `reject_whitespace_context_kind` — `"   "` should fail
   - `reject_empty_context_source` — `""` should fail
   - `reject_empty_content_context_item` — `{"content":"","tokens":10}` should fail
   - `reject_invalid_budget_target_exceeds_max` — `{"max_tokens":100,"target_tokens":200,...}` should fail
   - `reject_invalid_budget_negative_max` — negative max_tokens should fail
   - `reject_invalid_quota_require_exceeds_cap` — require > cap should fail
   - `reject_invalid_quota_negative_require` — require = -1.0 should fail

3. **Unknown field rejection tests**:
   - `reject_unknown_field_overflow_strategy` — e.g. `{"Throw": null, "extra": true}` (note: enum, so test a struct type)
   - `reject_unknown_field_context_item` — extra field in JSON object
   - `reject_unknown_field_context_budget` — extra field in JSON object
   - `reject_unknown_field_quota_entry` — extra field in JSON object

4. **Default handling tests for ContextItem**:
   - `context_item_defaults` — deserialize `{"content":"hello","tokens":10}`, verify:
     - kind is "Message" (default)
     - source is "Chat" (default)
     - priority is None
     - tags is empty vec
     - metadata is empty map
     - timestamp is None
     - future_relevance_hint is None
     - pinned is false
     - original_tokens is None

5. **Wire format tests** (verify exact JSON shape):
   - `context_kind_serializes_as_bare_string` — not wrapped in an object
   - `overflow_strategy_serializes_as_pascal_case` — `"Throw"`, `"Truncate"`, `"Proceed"`
   - `context_budget_reserved_slots_string_keys` — HashMap keys are string, not numeric

**Implementation notes:**
- Use `serde_json::to_string` and `serde_json::from_str` for all tests
- For ContextItem equality checks: compare field-by-field using accessor methods (PartialEq may not cover all semantic equality)
- For timestamp roundtrip: use a fixed `DateTime<Utc>` like `Utc.with_ymd_and_hms(2025, 1, 15, 12, 0, 0).unwrap()`
- Import from `cupel::*` (it's an integration test)
