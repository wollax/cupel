<objective>
Implement CupelPipeline and PipelineBuilder: the fluent builder API and the fixed pipeline execution engine. CupelPipeline wires Classify -> Score -> Deduplicate -> Sort -> Slice -> Place in fixed order. The builder validates at Build() time and provides sensible defaults (GreedySlice, ChronologicalPlacer, dedup on).

Purpose: This is the core deliverable of Phase 5 -- the first end-to-end context selection pipeline. Every subsequent phase builds on this foundation.

Output: Two sealed classes (CupelPipeline, PipelineBuilder) with comprehensive TDD test coverage covering builder validation, pipeline stage ordering, pinned item bypass, deduplication, and budget adjustment.
</objective>

    CupelPipeline: sealed class with internal constructor.

    Fields:
    - readonly IScorer _scorer
    - readonly ISlicer _slicer
    - readonly IPlacer _placer
    - readonly ContextBudget _budget
    - readonly bool _deduplicationEnabled

    public ContextResult Execute(IReadOnlyList<ContextItem> items, ITraceCollector? traceCollector = null):
    1. var trace = traceCollector ?? NullTraceCollector.Instance;
    2. CLASSIFY: partition items into pinned[] and scoreable[]. Skip items with Tokens < 0 (trace the exclusion if trace.IsEnabled).
    3. VALIDATE PINNED BUDGET: sum pinned tokens. If pinnedTokens > _budget.MaxTokens - _budget.OutputReserve, throw InvalidOperationException.
    4. SCORE: for each scoreable item, call _scorer.Score(item, scoreable) to produce ScoredItem[]. Pass only scoreable items as the allItems parameter (not pinned).
    5. DEDUPLICATE (if _deduplicationEnabled): Dictionary<string, int>(StringComparer.Ordinal). Key=Content, Value=index of best-scoring duplicate. Build deduped ScoredItem[] from dictionary values.
    6. SORT: stable sort scored items by score descending using (double Score, int Index) tuple array + Array.Sort with static delegate.
    7. SLICE: create adjusted ContextBudget with reduced TargetTokens and MaxTokens (subtract pinned tokens). Call _slicer.Slice(sortedScored, adjustedBudget, trace).
    8. RE-ASSOCIATE SCORES: build HashSet<ContextItem>(ReferenceEqualityComparer.Instance) from slicer output. Filter the sorted scored array to only items in the hashset, producing ScoredItem[] for the placer.
    9. MERGE PINNED: create ScoredItem[] for pinned items with Score=1.0. Concatenate with sliced scored items.
    10. PLACE: call _placer.Place(merged, trace).
    11. BUILD RESULT: return new ContextResult { Items = placed, Report = ... }.

    For the Report: if traceCollector is DiagnosticTraceCollector, populate Report. Otherwise null. (Check existing SelectionReport shape for exact construction.)

    Budget adjustment: create new ContextBudget(
        maxTokens: Math.Max(0, _budget.MaxTokens - _budget.OutputReserve - pinnedTokens) + _budget.OutputReserve,
        targetTokens: Math.Max(0, _budget.TargetTokens - pinnedTokens),
        outputReserve: _budget.OutputReserve,
        reservedSlots: _budget.ReservedSlots,
        estimationSafetyMarginPercent: _budget.EstimationSafetyMarginPercent
    )
    -- Actually simpler: the slicer uses TargetTokens for filling. Reduce TargetTokens by pinnedTokens. The MaxTokens ceiling also needs adjusting. Be careful with validation: the new TargetTokens must not exceed the new MaxTokens.

    Corrected budget adjustment:
    - effectiveMax = _budget.MaxTokens - _budget.OutputReserve - pinnedTokens (already validated >= 0)
    - effectiveTarget = Math.Max(0, _budget.TargetTokens - pinnedTokens)
    - effectiveTarget = Math.Min(effectiveTarget, effectiveMax) (ensure target <= max)
    - new ContextBudget(maxTokens: effectiveMax, targetTokens: effectiveTarget)
    -- Note: OutputReserve is already subtracted from effectiveMax, so set outputReserve=0 on the adjusted budget. ReservedSlots carries through.

    public async Task<ContextResult> ExecuteAsync(IContextSource source, ITraceCollector? traceCollector = null, CancellationToken cancellationToken = default):
    1. ArgumentNullException.ThrowIfNull(source)
    2. var items = await source.GetItemsAsync(cancellationToken).ConfigureAwait(false)
    3. return Execute(items, traceCollector)
