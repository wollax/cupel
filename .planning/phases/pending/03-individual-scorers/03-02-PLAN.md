    KindScorerTests (namespace Wollax.Cupel.Tests.Scoring):
    - DefaultWeights_SystemPrompt_ReturnsOne: SystemPrompt kind → 1.0
    - DefaultWeights_Message_ReturnsPointTwo: Message kind → 0.2
    - DefaultWeights_Ordinal: SystemPrompt > Memory > ToolOutput > Document > Message (ordinal chain)
    - UnknownKind_ReturnsZero: custom ContextKind not in map → 0.0
    - CustomWeightMap_OverridesDefaults: constructor-injected map used instead of defaults
    - IgnoresAllItems: score depends only on item.Kind, not on allItems

    TagScorerTests (namespace Wollax.Cupel.Tests.Scoring):
    - EmptyTags_ReturnsZero: item with no tags → 0.0
    - SingleMatchingTag_ReturnsNormalized: one matching tag → that tag's weight / total weight
    - MultipleMatchingTags_SumsNormalized: two matching tags → sum of weights / total weight
    - NoMatchingTags_ReturnsZero: item tags don't match any configured tags → 0.0
    - AllTagsMatch_ReturnsOne: every configured tag present → 1.0
    - PartialMatch_ScoresLower_ThanFullMatch: item with 1 tag < item with all tags (ordinal)
    - UnknownTagsIgnored: tags on item not in weight map are simply not counted
    - Constructor_RequiresWeightMap: TagScorer has no parameterless constructor — requires IReadOnlyDictionary<string, double>
  </behavior>
  <implementation>
    KindScorer — sealed class in Wollax.Cupel.Scoring:
    - Private static readonly DefaultWeights dictionary: SystemPrompt=1.0, Memory=0.8, ToolOutput=0.6, Document=0.4, Message=0.2
    - Parameterless constructor chains to overload with DefaultWeights
    - Constructor(IReadOnlyDictionary<ContextKind, double> weights) stores reference
    - Score(): _weights.TryGetValue(item.Kind, out var weight) ? weight : 0.0
    - See RESEARCH.md Pattern 2 for exact code example

    TagScorer — sealed class in Wollax.Cupel.Scoring:
    - Constructor(IReadOnlyDictionary<string, double> tagWeights) — required, no parameterless constructor
    - Pre-compute _totalWeight in constructor: for-loop summing all values (avoid LINQ .Sum())
    - Score(): for-loop over item.Tags, TryGetValue each tag in _tagWeights, accumulate matchedSum. Return _totalWeight == 0 ? 0.0 : matchedSum / _totalWeight
    - Tag lookup should use the dictionary's default comparer. Recommend constructing with StringComparer.OrdinalIgnoreCase if case-insensitive behavior is desired, but TagScorer itself just uses TryGetValue.
  </implementation>

  <behavior>
    FrequencyScorerTests (namespace Wollax.Cupel.Tests.Scoring):
    - EmptyTags_ReturnsZero: item with no tags → 0.0
    - SingleItem_ReturnsZero: allItems has only one item → 0.0 (no peers to compare)
    - AllItemsShareTag_ReturnsOne: every other item shares a tag → 1.0
    - NoItemsShareTag_ReturnsZero: no other item shares any tag → 0.0
    - PartialOverlap_ScoresBetweenZeroAndOne: some items share tags, some don't → score in (0, 1)
    - CaseInsensitive_TagComparison: "API" matches "api" (OrdinalIgnoreCase)
    - SelfExclusion: item does not count itself as a match (uses ReferenceEquals)
    - MoreSharedItems_ScoresHigher: item sharing tags with 3/4 peers scores higher than item sharing with 1/4 (ordinal)
    - MultipleSharedTags_CountsOnce: sharing 2 tags with same item still counts as 1 match (not 2)
  </behavior>
  <implementation>
    FrequencyScorer — sealed class in Wollax.Cupel.Scoring:
    - Parameterless constructor (no configuration needed)
    - Score():
      1. If item.Tags.Count == 0 || allItems.Count <= 1 → return 0.0
      2. For-loop over allItems: skip self (ReferenceEquals), skip empty tags, call SharesAnyTag helper
      3. Return matchingItems / (double)(allItems.Count - 1)
    - Private static SharesAnyTag(IReadOnlyList<string> a, IReadOnlyList<string> b) → bool:
      Nested for-loops with string.Equals(a[i], b[j], StringComparison.OrdinalIgnoreCase)
    - See RESEARCH.md Pattern 3 for exact code example
